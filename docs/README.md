# Documentation

This folder holds the durable reference material for `profile-links-starter`.
The goal is to explain what the starter is supposed to do, how it is structured, and what must stay true as the project evolves.

Start with the root [README](../README.md) for onboarding, installation, content editing, and deployment steps.
Use the documents in this folder when you need design intent, architecture context, or a maintenance checklist.

## Document Map

- [user-stories.md](./user-stories.md) defines the product goals, supported audiences, and acceptance signals.
- [architecture.md](./architecture.md) explains the config-first runtime model, route structure, and system boundaries.
- [config-reference.md](./config-reference.md) maps each editable content file to the feature it drives.
- [tasks.md](./tasks.md) provides a concise checklist for implementation, review, and release readiness.

## Scope

This docs set is intentionally small.
It focuses on stable project knowledge, not historical build notes or one-time planning artifacts.

Internal tool instructions and agent-specific notes belong in their own dedicated files outside this folder.
