# User Stories

## Visitor Stories
- As a visitor, I can see one clear profile page with the person name, short bio, and main links.
- As a visitor, I can open social and contact links without needing to search the site.
- As a visitor, I can read the legal pages from direct routes when I need them.
- As an external site owner, I can reuse the badge or list view through dedicated embed routes.

## Fork Owner Stories
- As a fork owner, I can replace the example `Your Name` content by editing JSON config files.
- As a fork owner, I can switch the site language in one place so SEO and page metadata stay aligned.
- As a fork owner, I can deploy the starter to GitHub Pages with minimal setup.
- As a fork owner, I can keep the same visual style while changing the content for a new person or brand.
- As a fork owner, I can embed the list or badge views elsewhere without rebuilding the app structure.

## Search And AI Stories
- As a search engine, I can read a stable title, description, canonical URL, and language from config.
- As an AI crawler, I can find structured profile data, link targets, and clear page summaries.
- As a compliance crawler, I can reach `contact`, `privacy`, `license`, `gtc`, and `impressum` on their own pages.
