# Project Plan

## Human Plan

1. Inspect the reference site and identify only the files that control the visible result.
2. Create a new Nuxt starter in `profile-links-starter`.
3. Copy the visual layer that matters and remove multi-language runtime complexity.
4. Move all user-editable data into `config/` with one file per major content area.
5. Rebuild the homepage with example `Your Name` data.
6. Add metadata, structured data, robots, sitemap, and llms generation.
7. Add direct legal routes and keep the homepage legal access easy.
8. Write short docs for stories, tasks, setup, and deployment.
9. Verify the static build and note any remaining risks.

## Done Definition

- The starter renders from config only.
- One language setting drives SEO.
- GitHub Pages deployment is documented.
- Example content is complete enough for a fork owner to replace quickly.
