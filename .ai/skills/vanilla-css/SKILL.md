# Vanilla CSS

Use this skill for styling work that should remain framework-light and maintainable.

## Default Approach

1. Start with layout, spacing, and typography before decorative flourishes.
2. Use CSS custom properties for reusable tokens.
3. Prefer component-scoped styles and predictable naming.
4. Build mobile-first and verify larger breakpoints intentionally.

## Project Defaults

- keep the visual system expressive but not heavy
- preserve the warm paper backdrop, oversized hero rhythm, rounded panels, and profile-card feel unless the task explicitly asks for redesign
- keep list, tile, and badge views visually related, not like unrelated mini-products
- avoid utility sprawl and deeply nested selectors
- make hover, focus, and reduced-motion states intentional
